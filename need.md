# 一、整体要求

* 面向移动端（手机小程序场景）
* 技术栈：Vue（建议Composition API或简化Options API），避免引入重量级库
* 状态管理：轻量（优先使用本地state / composables / 简单store，不使用Vuex/Pinia除非必要）
* UI按设计稿还原，但允许合理简化
* 数据优先使用本地mock（JSON），后续可扩展API
* 所有页面需具备基本交互逻辑（不是纯UI）

# 二、核心页面及逻辑

## 1. 首页（Home）

功能目标：快速开始训练 + 激励用户

需要实现：

* 获取“今日训练计划”（本地mock或缓存）
* 展示：

  * 今日训练卡片（训练名称、预计时长、难度）
  * 连续打卡天数（streak）
  * 激励文案（随机）
* 主按钮：

  * 「开始训练」→ 跳转训练执行页

状态逻辑：

* 如果今日未训练 → 显示“开始训练”
* 如果已完成 → 显示“已完成 + 再练一次”

---

## 2. 训练执行页（Workout）

这是核心页面，必须可用

需要实现：

* 动作列表（数组驱动）
  每个动作包含：

  * name
  * duration / reps
  * restTime
  * demo（图片或占位）

页面功能：

* 当前动作高亮
* 倒计时（setInterval实现）
* 当前组数（如 2/4）
* 操作按钮：

  * 完成当前动作 → 进入下一个
  * 跳过
* 自动流转：

  * 动作完成 → 休息 → 下一个动作

状态管理：

* currentExerciseIndex
* currentSet
* timer
* workoutStatus（进行中 / 完成）

结束逻辑：

* 全部完成 → 跳转打卡页
* 写入训练记录（本地存储）

---

## 3. 打卡页（Check-in）

功能目标：强化用户反馈

需要实现：

* 显示：

  * 今日完成训练 ✔
  * 消耗时间
  * 简单反馈（如“干得不错🔥”）
* 打卡按钮：

  * 点击后：

    * 更新 streak（连续天数）
    * 写入 localStorage

可选增强：

* 简单动画（完成反馈）

---

## 4. 数据页（Stats）

需要实现：

* 从本地读取：

  * 训练记录
  * 体重数据（可mock）
* 展示：

  * 训练次数统计
  * streak
* 图表：

  * 可用简单div模拟，不强制引入图表库（轻量优先）

---

## 5. 我的页面（Profile）

需要实现：

* 用户信息（mock）
* 当前目标（增肌 / 减脂）
* 设置入口（可空实现）
* 成就（可简单展示）
